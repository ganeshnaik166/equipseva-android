## Critical issues (must fix before launch)

- [ ] **website/index.html:21, :49 — og.png and logo.png referenced but not in repo**
  `<meta property="og:image" content="https://equipseva.com/og.png" />` (line 21) and the
  `Organization.logo` in JSON-LD (line 49) both point to files that do not exist.
  Every social share shows a broken image; Google Rich Results will reject the logo.
  Fix: create `website/og.png` (1200×630 px) and `website/logo.png` (512×512 px) and commit both.

- [ ] **website/contact.html:45 — contact form does not submit**
  The form's `onsubmit` handler only calls `event.preventDefault()` and changes the button
  text to "Sent ✓" — no data is ever sent anywhere. Users believe their message was delivered
  but it is silently discarded.
  Fix: replace the inline handler with a real endpoint (Formspree, Web3Forms, Netlify Forms,
  or a custom API route); add server-side validation and a visible error state.

- [ ] **website/styles.css:452 — focus outline removed on form inputs (WCAG 2.4.7 Level AA)**
  `.form input:focus, .form textarea:focus, .form select:focus { outline: 0; … }` suppresses
  the browser's default focus ring. The replacement `border-color` change is the only focus
  indicator — it is not reliably visible at all zoom levels and fails keyboard-only users.
  Fix: replace `outline:0` with `outline: 2px solid rgba(0,211,192,.8); outline-offset: 2px`.

- [ ] **All 17 pages — no skip-to-content link (WCAG 2.4.1 Level AA)**
  No page provides a way for keyboard or screen-reader users to jump past the navigation to
  `<main>`. Every page load forces them to tab through 8 nav links and a CTA button first.
  Fix: add as the very first child of `<body>` on every page:
  ```html
  <a href="#top" class="skip-link">Skip to main content</a>
  ```
  and in `styles.css`:
  ```css
  .skip-link { position:absolute; transform:translateY(-100%); }
  .skip-link:focus { transform:translateY(0); }
  ```
  (`<main id="top">` already exists on index.html; verify other pages have matching IDs.)

- [ ] **website/styles.css — no :focus-visible styles for interactive elements outside the form**
  Nav links (`.nav-links a`), the burger button (`.burger`), and all CTA buttons (`.btn`) have
  no `:focus` or `:focus-visible` rule, so keyboard users see no visual cue when tabbing to them.
  Fix: add to `styles.css`:
  ```css
  .nav-links a:focus-visible, .btn:focus-visible, .burger:focus-visible,
  .brand:focus-visible, .mobile-menu a:focus-visible {
    outline: 2px solid var(--teal);
    outline-offset: 3px;
    border-radius: 4px;
  }
  ```

### SEO — Open Graph & Twitter cards

- [ ] **16 of 17 pages — og:image missing** (only index.html has it)
  Every page that gets shared on LinkedIn, WhatsApp, or Twitter will render with no image.
  Fix: once `og.png` is created (see Critical #1), add to the `<head>` of all 16 remaining pages:

- [ ] **16 of 17 pages — twitter:card / twitter:description missing**
  Same scope as above. Also add `<meta name="twitter:description" content="…" />` matching the

### SEO — Heading hierarchy

WCAG 1.3.1 / Google's crawler both expect headings to nest without skipping levels (h1 → h2 → h3).

- [ ] **website/features.html:61 — h3 cards with no preceding h2**
  After the h1 page title, 12 feature `<article>` cards use `<h3>` with no intervening `<h2>`.
  The only h2 on the page is the bottom CTA ("Ready to fix it faster?").
  Fix: add a visually styled (or visually hidden) `<h2>` as the section heading before the card grid,
  e.g. `<h2 class="reveal">Platform capabilities</h2>`, or change card titles to `<strong>`.

- [ ] **website/how-it-works.html:45 — h3 step items with no preceding h2**
  Same pattern: h1 → four `<h3>` step headings → h2 CTA.
  Fix: add a section `<h2>` before the step list, e.g. `<h2>The four steps</h2>`.

- [ ] **website/pricing.html:37 — h3 plan cards with no preceding h2**
  h1 → three `<h3>` plan cards → h2 "Pricing questions".
  Fix: add `<h2>Plans</h2>` (or visually hidden) before the card row.

- [ ] **website/contact.html:39 — h3 channel cards, no h2 anywhere**
  h1 "Let's talk." → four `<h3>` contact-channel cards. No h2 at all on the page.
  Fix: add `<h2>Ways to reach us</h2>` (or equivalent) before the channel grid.

- [ ] **website/blog/index.html:38 — h3 post titles, no h2 anywhere**
  h1 "The EquipSeva Blog." → three `<h3>` post title cards. No h2.
  Fix: add `<h2>Latest posts</h2>` before the post list.

### Structured data (JSON-LD)

- [ ] **8 pages — no JSON-LD at all**
  `for-hospitals.html`, `for-engineers.html`, `pricing.html`, `about.html`,
  `blog/index.html`, `privacy.html`, `terms.html`, `refund.html` have no structured data.
  At minimum each needs a `BreadcrumbList`; `for-hospitals.html` and `for-engineers.html`
  would also benefit from a `Service` schema; `about.html` should carry an `Organization`
  or `Person` block.

- [ ] **3 blog posts — BlogPosting JSON-LD missing required fields**
  `blog/hospital-equipment-maintenance-india.html:14–16`,
  `blog/biomedical-engineer-career-india.html:14–16`,
  `blog/kyc-verified-engineers-trust.html:14–16` are each missing `"url"`, `"description"`,
  and `"image"` — all recommended by Google for article rich results.
  Fix example for first post:
  ```json
  "url": "https://equipseva.com/blog/hospital-equipment-maintenance-india.html",
  "description": "Why a single hour of OT light failure cascades into theatre rescheduling …",
  "image": "https://equipseva.com/og.png"
  ```

### Crawl / sitemap

- [ ] **website/sitemap.xml — no `<lastmod>` dates**
  All 16 `<url>` blocks lack `<lastmod>`. Search engines use this hint to prioritise
  re-crawling recently changed pages.
  Fix: add `<lastmod>2026-05-01</lastmod>` (or the per-file commit date) to each entry.

### Accessibility — misc

- [ ] **3 blog posts — no `<time datetime="">` in HTML**
  Publish dates live only in JSON-LD. Assistive technologies and feed readers benefit from
  a machine-readable date in the document itself.
  Fix: wrap each date string in e.g. `<time datetime="2026-04-12">2026-04-12</time>`.

### Browser compatibility

- [ ] **All 17 pages — inline SVG `data:` favicon, no `.ico` fallback**
  The `<link rel="icon" href="data:image/svg+xml,…">` favicon works in Firefox/Chrome/Safari
  but is ignored by some Windows browsers and many RSS/bookmark tools that expect a `.ico` file.
  Fix: add `website/favicon.ico` (32×32 multi-size) and a second `<link>` tag:
  ```html
  <link rel="icon" sizes="any" href="/favicon.ico" />
  <link rel="icon" type="image/svg+xml" href="data:image/svg+xml,…" />
  ```
